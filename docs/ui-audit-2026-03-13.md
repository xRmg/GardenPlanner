# Garden Planner — UI Quality Audit (v2)

> **Audited**: 2026-03-13 (v2 — full re-audit with fresh component reads)
> **Scope**: All feature components + core layout (`app/components/`, `app/App.tsx`, `styles/`)  
> **Method**: Static code analysis against WCAG 2.1 AA, frontend-design anti-pattern checklist, and project design context (`CLAUDE.md`)

---

## Anti-Patterns Verdict

**Verdict: PASS with cautions.** The interface does not look generically AI-generated. The vibrant garden green palette, physical grid metaphor, and compact data-dense layout are distinctive and purposeful. However, several specific anti-pattern fingerprints are present:

| Tell | Where | Verdict |
|---|---|---|
| `backdrop-blur` + `bg-white/X` glass cards *everywhere* | All panels, dialogs, cards, EventsBar, ToolBar | ⚠ Overused — 17 instances |
| Decorative `blur-2xl` rounded blobs | `App.tsx`, `AppErrorBoundary.tsx` | ⚠ Non-functional decoration |
| Identical card grid (icon + heading + badges, repeated) | Plants tab, Seedlings tab | ⚠ Same-sized card grid anti-pattern |
| `animate-in zoom-in` on every grid mount | `PlanterGrid.tsx` | Borderline — needs reduced-motion guard |
| `hover:scale-105 active:scale-95` on buttons | Buttons throughout | Minor — eases cleanly |

## Executive Summary

| Severity | Count |
|---|---|
| Critical | 3 |
| High | 7 |

**Top 5 most critical issues:**
1. `alert()` used for form validation in dialogs — blocks keyboard users and is inaccessible
2. No `prefers-reduced-motion` support anywhere in the app — all animations always fire
3. Icon-only buttons lack `aria-label` — invisible to screen readers
4. `PlantDetailsDialog` and `PlanterDialog` use raw `gray-*` / `blue-*` color classes, diverging from the design token system
5. The app is not usable below ~900px viewport width — no mobile layout exists

## Detailed Findings

### Critical Issues

---

**[C-1] `alert()` used for form validation**

- **Location**: `app/components/PlanterDialog.tsx` (lines 88, 107, 118) · `app/components/PlantDefinitionDialog.tsx` (line 274)
- **Severity**: Critical
- **Category**: Accessibility
- **Description**: Native browser `alert()` is called when form validation fails (empty planter name, invalid virtual section range).
- **Impact**: Blocks the event loop; inaccessible to screen readers using dynamic content announcements; inconsistent with the app's own Sonner toast/notification system; breaks the design system entirely with a raw OS dialog.
- **WCAG**: 4.1.3 Status Messages (AA)
- **Recommendation**: Replace all `alert()` calls with inline validation messages rendered next to the relevant field, or use the existing Sonner toast infrastructure for one-off non-blocking errors.
- **Suggested skill**: `/harden`

---

**[C-2] No `prefers-reduced-motion` support anywhere**

- **Location**: `styles/` (no CSS media query) · `PlanterGrid.tsx` (`animate-in zoom-in duration-500`) · `App.tsx` (`transition-transform duration-300`) · all `transition-all` usages
- **Severity**: Critical
- **Category**: Accessibility / Motion
- **Description**: Zero instances of `@media (prefers-reduced-motion: reduce)` or Tailwind's `motion-reduce:` modifier exist anywhere across component code or global CSS. Every animation fires unconditionally.
- **Impact**: Users with vestibular disorders, epilepsy triggers, or system reduced-motion preferences receive no accommodation.
- **WCAG**: 2.3.3 Animation from Interactions (AAA) / 2.3.1 (AA for seizure risk)
- **Recommendation**: Add `@media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } }` to `styles/index.css` as a global safety net. Additionally add `motion-reduce:` Tailwind modifiers to the most prominent animations (`animate-in`, `zoom-in`, `fade-in`).
- **Suggested skill**: `/animate`

---

**[C-3] The app has no mobile layout — unusable below ~900px**

- **Location**: `app/App.tsx` — the entire layout uses `flex flex-row` with a fixed `w-72` EventsBar always visible
- **Severity**: Critical
- **Category**: Responsive Design
- **Description**: The main layout is a horizontal flex with a permanent `w-72` sidebar. There are no `sm:` or `md:` responsive breakpoints on the main layout flex. The ToolBar wraps but overlaps content. The planter grid (`inline-block`) overflows horizontally.
- **Impact**: The app is not usable on phones or tablets below ~900px horizontal resolution.
- **Recommendation**: At minimum, hide the EventsBar on small screens behind a slide-out sheet/drawer. Make the ToolBar collapse into a menu at narrow widths. The `PlanterGrid` grid cells need to scale.
- **Suggested skill**: `/adapt`

### High-Severity Issues

---

**[H-1] Icon-only buttons missing `aria-label`**

- **Location**: `app/components/PlanterGrid.tsx` — move up/down, edit, delete buttons use `title=""` only · `app/App.tsx` — area move/delete buttons · Settings API key visibility toggle
- **Severity**: High
- **Category**: Accessibility
- **Description**: Buttons containing only icon SVGs have `title` attributes for tooltip-style hints but `title` is not reliably announced by screen readers. They need `aria-label`.
- **Impact**: Screen reader users cannot determine the purpose of icon-only controls (edit, delete, move, toggle visibility).
- **WCAG**: 4.1.2 Name, Role, Value (AA)
- **Recommendation**: Add `aria-label="Edit planter"`, `aria-label="Delete planter"`, etc. alongside existing `title` attributes.
- **Suggested skill**: `/harden`

---

**[H-2] Form inputs lack `<label>` associations throughout**

- **Location**: `app/components/PlanterDialog.tsx` — rows/cols inputs · `app/components/PlantDetailsDialog.tsx` — variety and harvest date inputs · Settings section in `App.tsx` (location, API key, model inputs all have adjacent `<label>` elements not bound via `htmlFor`)
- **Severity**: High
- **Category**: Accessibility
- **Description**: Visual labels are present but not bound to their inputs via `htmlFor`/`id` pairs. Only 2 `htmlFor` uses exist in feature code — both in `PlantDefinitionDialog.tsx`.
- **Impact**: Screen readers cannot associate labels with inputs. Clicking a label does not focus the input.
- **WCAG**: 1.3.1 Info and Relationships (A) · 4.1.2 Name, Role, Value (AA)
- **Recommendation**: Add `id` attributes to all `<input>` elements and corresponding `htmlFor` to their `<label>` elements.
- **Suggested skill**: `/harden`

---

**[H-3] `PlantDetailsDialog` and `PlanterDialog` use hard-coded raw color classes**

- **Location**: `app/components/PlantDetailsDialog.tsx` — `text-gray-500`, `text-gray-600`, `border-gray-300`, `focus:ring-blue-500`, `bg-blue-600` · `app/components/PlanterDialog.tsx` — same pattern, save button uses `bg-blue-600`
- **Severity**: High
- **Category**: Theming
- **Description**: These dialogs were written with raw Tailwind gray and blue utilities instead of the design token CSS variables (`text-foreground`, `text-muted-foreground`, `border-border`, `ring-primary`, `bg-primary`). Visual inconsistency vs. the rest of the app, and dark mode will not work for these components.
- **Recommendation**: Replace all `gray-*` utilities with `foreground`/`muted-foreground`/`border` tokens, and replace `blue-*` with `primary`.
- **Suggested skill**: `/normalize`

---

**[H-4] Interactive grid squares (`PlanterGrid` cells) have no keyboard support**

- **Location**: `app/components/PlanterGrid.tsx` — the `onClick` cell `<div>` elements
- **Severity**: High
- **Category**: Accessibility
- **Description**: The garden grid cells are `<div>` elements with `onClick` handlers. They are not keyboard-focusable, do not respond to Enter/Space, and have no `role="button"` or `tabIndex`.
- **Impact**: Keyboard-only users and screen reader users cannot interact with the core feature of the app — placing and removing plants.
- **WCAG**: 2.1.1 Keyboard (A) — a Level A failure
- **Recommendation**: Either change grid cells to `<button type="button">` elements with appropriate `aria-label` (e.g., "Row 1, Column 2 — empty"), or add `role="button"`, `tabIndex={0}`, and `onKeyDown` handlers.
- **Suggested skill**: `/harden`

---

**[H-5] "My Plants / Seeds" tab uses the `<Users>` icon — semantically wrong**

- **Location**: `app/App.tsx` line 384
- **Severity**: High
- **Category**: Accessibility / UX Writing
- **Description**: The "My Plants / Seeds" tab uses the `<Users>` (person silhouettes) Lucide icon, which semantically communicates "people" or "teammates", not "plants".
- **Impact**: Screen reader users and icon-dependent users will be confused.
- **WCAG**: 1.3.3 Sensory Characteristics (AA)
- **Recommendation**: Replace with `<Leaf>` or `<Sprout>` — both already imported in this file.
- **Suggested skill**: `/clarify`

---

**[H-6] Glassmorphism overuse — 17 instances of `backdrop-blur` across the app**

- **Location**: Every content panel, every dialog, ToolBar, EventsBar, every card in the Plants/Seedlings tabs
- **Severity**: High
- **Category**: Anti-Patterns
- **Description**: `backdrop-blur` is applied uniformly to every surface with no hierarchy of depth. When everything blurs, nothing does. This is the "glassmorphism everywhere" anti-pattern.
- **Impact**: GPU rendering pressure for no visual gain; sameness that undermines technical-precision aesthetic.
- **Recommendation**: Reserve `backdrop-blur` for one or two intentional surfaces (floating ToolBar, overlay dialogs). Remove from content panels and card grids.
- **Suggested skill**: `/distill`

---

**[H-7] "New Planter" dashed button has critically low contrast**

- **Location**: `app/App.tsx` line ~1540: `border-gray-200/50 text-gray-300`
- **Severity**: High
- **Category**: Accessibility / Contrast
- **Description**: The "New Planter" add button uses `text-gray-300` (extremely low contrast light gray on white background). Estimated contrast ratio ~1.5:1.
- **Impact**: Low-vision users cannot see this affordance at all before hovering.
- **WCAG**: 1.4.3 Contrast (Minimum) (AA) — requires 4.5:1 for text
- **Recommendation**: Use `border-border/40 text-muted-foreground` to achieve a visible but subtle default state.
- **Suggested skill**: `/normalize`

### Medium-Severity Issues

---

**[M-1] Decorative background blobs serve no functional purpose**

- **Location**: `app/App.tsx` lines 362–363 · `app/components/AppErrorBoundary.tsx` lines 49–50
- **Severity**: Medium
- **Category**: Anti-Patterns
- **Description**: Two absolute-positioned `blur-2xl` rounded `<div>` blobs serve as purely decorative background elements. At `bg-primary/5` opacity they're barely perceptible but add render complexity.
- **Recommendation**: Remove. The clean `bg-background` is sufficient. A subtle `bg-gradient-to-br from-background to-muted/30` achieves atmosphere without blobs.
- **Suggested skill**: `/distill`

---

**[M-2] `text-[9px]` and `text-[8px]` used in 25+ places — below legible minimum**

- **Location**: `app/components/CalendarView.tsx`, `PlanterGrid.tsx`, `ToolBar.tsx`, `App.tsx` — ~25+ instances
- **Severity**: Medium
- **Category**: Accessibility
- **Description**: 9px and 8px text is below the effective minimum legible size. Seedling cards have 8px labels. Even with `uppercase tracking-wider` compensation, this remains a concern at scale.
- **WCAG**: 1.4.4 Resize Text (AA)
- **Recommendation**: Establish a minimum of `text-xs` (12px) for label/chip text. Extract a shared `<MicroLabel>` component.
- **Suggested skill**: `/polish`

---

**[M-3] Dark mode token palette loses the garden green brand color**

- **Location**: `styles/theme.css` (dark mode block)
- **Severity**: Medium
- **Category**: Theming
- **Description**: In the `.dark` block, `--primary` is `oklch(0.985 0 0)` (near white) and `--secondary` is `oklch(0.269 0 0)` (dark gray). The vibrant garden green (#22c55e) is entirely lost in dark mode.
- **Impact**: If dark mode is ever activated (system pref or future toggle), the entire brand identity disappears — buttons become white.
- **Recommendation**: Replace dark mode `--primary` with a lightened garden green (e.g., `oklch(0.72 0.2 145)`) for brand continuity on dark surfaces.
- **Suggested skill**: `/colorize`

---

**[M-4] EventsBar `<h2>` heading renders no accessible text**

- **Location**: `app/components/EventsBar.tsx` line ~255
- **Severity**: Medium
- **Category**: Accessibility / UX Writing
- **Description**: The `<h2>` element in EventsBar's header renders only a `<Calendar>` icon with no accompanying text. The icon has no `aria-hidden` and provides no label text for screen readers.
- **WCAG**: 1.3.1 Info and Relationships (A)
- **Recommendation**: Add visible or visually-hidden text: `<h2><Calendar aria-hidden="true" /> <span>Activity</span></h2>`.
- **Suggested skill**: `/clarify`

---

**[M-5] `zoom-in` entrance animation on `PlanterGrid` fires on every mount, no motion guard**

- **Location**: `app/components/PlanterGrid.tsx` line 276 — `animate-in fade-in zoom-in duration-500`
- **Severity**: Medium
- **Category**: Performance / Motion
- **Description**: `animate-in zoom-in` fires every time the component mounts — which can happen on parent re-render for multiple planters. No `motion-reduce:` guard.
- **Recommendation**: Add `motion-reduce:animate-none` Tailwind modifier or gate with `@media (prefers-reduced-motion)`.
- **Suggested skill**: `/animate`

---

**[M-6] Plant and Seedling tabs use an identical same-sized uniform card grid**

- **Location**: `app/App.tsx` — Plants tab and Seedlings tab
- **Severity**: Medium
- **Category**: Anti-Patterns
- **Description**: Both tabs render identical `grid-cols-1 md:grid-cols-3 lg:grid-cols-4` grids of uniform cards with emoji icon, heading, badges, and data rows. This is the "identical card grid" anti-pattern — monotonous visual rhythm.
- **Recommendation**: Introduce visual variation: feature cards for "sow now" plants, compact list variant for browsing, variable row heights.
- **Suggested skill**: `/bolder`

---

**[M-7] Disabled "bundled plant" settings button has no accessible explanation**

- **Location**: `app/App.tsx` line ~904 — `<button className="... cursor-not-allowed opacity-30">` with `<SettingsIcon>` only
- **Severity**: Medium
- **Category**: Accessibility / UX Writing
- **Description**: This button is visually disabled but has no `disabled` attribute, no `aria-label`, and no tooltip explaining why bundled plants can't be edited. Keyboard users can still activate it.
- **WCAG**: 4.1.2 Name, Role, Value (AA)
- **Recommendation**: Add `disabled` attribute + `aria-label="Bundled plants cannot be edited"` or a tooltip.
- **Suggested skill**: `/harden`

## Patterns & Systemic Issues

1. **Glassmorphism uniformity** — `backdrop-blur` + `bg-white/X` + `border-white/Y` applied to every surface without discrimination. 17 instances. Needs a deliberate "when to blur and when not to" decision.

2. **Hard-coded color drift in older dialogs** — `PlantDetailsDialog` and `PlanterDialog` were written before the design token system solidified. They use raw `gray-*` and `blue-*` utilities while newer components use CSS custom properties.

3. **No `aria-label` discipline on icon-only buttons** — `title=""` is used as a proxy throughout. Needs to be replaced with `aria-label` consistently.

4. **No reduced-motion support whatsoever** — neither a CSS media query safety net nor any Tailwind `motion-reduce:` modifiers exist anywhere.

5. **Sub-12px text as a recurring pattern** — `text-[8px]` and `text-[9px]` appear 20+ times as a design shortcut for cramming labels into small spaces.

## Positive Findings

- ✅ **Consistent design token use** in new components — `App.tsx`, `EventsBar.tsx`, `CalendarView.tsx` all use `text-foreground`, `bg-primary`, `text-muted-foreground` properly.
- ✅ **Radix UI dialogs** provide correct focus trapping, `aria-modal`, and keyboard dismiss (Escape) out of the box for all dialogs.
- ✅ **`shadcn/ui` Button component** has proper `focus-visible` ring and `aria-invalid` support.
- ✅ **Sonner toast system** correctly integrated for async errors — avoids `alert()` in async paths.
- ✅ **`DialogDescription`** used in all dialogs, correctly providing accessible description text.
- ✅ **Semantic heading hierarchy** broadly respected — H1 per tab, H2 for sections, H3 for cards.
- ✅ **`lang="en"`** correctly set on the HTML document.
- ✅ **Empty states** are informative and teach the interface (Areas, Seedlings, Plants tabs).
- ✅ **`animate-pulse`** only used on the "Saved" dot indicator — appropriate and non-distracting.
